## 1 各部件详解与自检清单

文件落位：`07_paper/sections/01_metadata.md … 07_references.md`。一次一个部件，自检全过再下一个。

### Step 1 元数据 → `01_metadata.md`

结构：
```markdown
# [论文题名，中文≤20字]

作者1, 作者2, 通讯作者*

单位全称, 省份 城市 邮编

*通讯作者: 姓名, E-mail: xxx@xxx.edu.cn

【关键词】关键词1；关键词2；关键词3；关键词4；关键词5
```

自检：
- [ ] 题名 ≤ 20 个中文字符，结构 `[对象/地区] + [暴露/方法] + [结局] + [研究类型]`
- [ ] 关键词 3–8 个，分号分隔；≥2 个为 MeSH 规范词或研究核心词
- [ ] 题名不含非公认缩写、不含标点、无副标题

### Step 2 摘要 → `02_abstract.md`

结构式四段：`【目的】【方法】【结果】【结论】`。字数 300–500 中文字符。
- 【目的】1–2 句，不复述题名（≥30% 文字不同）。
- 【方法】2–4 句：研究设计 + 数据来源 + 核心分析方法 + 样本量。
- 【结果】3–5 句，含 ≥3 个关键数据（与源完全一致：估计值、95%CI、P 值）。
- 【结论】1–2 句，明确结论 + 建议。

自检：
- [ ] 恰好 4 个标签，字数 300–500
- [ ] 【结果】≥3 个具体数字，逐字来自 `0_result_summaries.md`
- [ ] 第三人称过去时；不引用图表与文献；缩写首次写全称
- [ ] 无"进行了……的研究/分析/评估"等填充短语

### Step 3 引言 → `03_introduction.md`

4 段漏斗式（宽→窄），总字数 800–1200：

| 段 | 功能 | 字数 |
|----|------|------|
| 1 | 背景与问题重要性（全局流行/政策/机制数据） | 200–300 |
| 2 | 既往研究与不足（综述非罗列，明确空白） | 250–350 |
| 3 | 本研究必要性（此地区/此人群/此方法的特殊性） | 150–250 |
| 4 | 研究目的（"本研究旨在"，收束至本研究） | 100–150 |

自检：
- [ ] 恰好 4 段，总字数 800–1200
- [ ] 第1段含领域流行/负担数据；第2段 ≥2 条既往不足；第4段含"本研究旨在"且与 Gap 对应
- [ ] `[待补充引用]` ≥5 处
- [ ] 不泄露本研究结果数据
- [ ] 段首不重复（"本"开头 ≤2 段）；AI 大词（见 chinese-anti-ai）≤2 次

### Step 4 方法 → `04_methods.md`

5–7 小节，总字数 1200–1800：
```
## 1 材料与方法
### 1.1 研究设计
### 1.2 研究对象 / 数据来源（纳入排除标准）
### 1.3 变量定义（暴露/自变量、结局、协变量）
### 1.4 [核心方法/模型/指标]
### 1.5 统计分析
### 1.6 伦理声明
```
要求：充分到可复现；公式用 `$$LaTeX$$`；所有参数取值来自源；写明软件版本与显著性水平。

#### 方法只写"做了什么"，不写"为什么/怎么决定的"（CRITICAL，最常翻车）

方法节是研究者陈述**已完成的操作**，不是分析日记、不是决策辩解、不是教科书。下列四类一律删除或压成
半句，**不要写进方法**：

| 不该写（信号词） | 为什么不写 | 改成 |
|------------------|------------|------|
| **动机/辩解**："因……易产生多重共线性""为避免……""鉴于……故采用""考虑到……" | 方法陈述做法，不论证选择 | 直接说"采用 X" |
| **排除理由**："就医方式因与三项几乎不相关、构念不同，故未纳入合成" | 排除决策属内部分析过程 | "就医方式作现状单独描述"（只说处置，不说为什么排） |
| **分析前/中的过程**："正式分析前作数据质量核查（配对完整性、逐行校验、缺失核查）""合成前作单维性核查""检出异方差后改用……" | 数据清洗与诊断流水账不进方法 | 删；诊断只在直接影响"报告哪个估计"时留半句（如"因检出异方差，区间估计以 Bootstrap 为准"） |
| **教科书定义**："MCA 是适用于多个分类变量的降维方法，将类别投影到低维空间，使共现频繁的类别靠近……" | 读者已懂方法学，不需科普 | "采用 MCA 对 12 个分类变量降维，取前三维度（累计解释 44.0%）"——一句话给做法+结果 |

经验：一段方法里出现"原因是 / 因 / 为了 / 鉴于 / 故 / 是一种……的方法 / 分析前作了……核查"就是写跑偏了。
**对照：合成得分该怎么写**——只写"将 A、B、C 三条目赋分相加构成可及性得分（0–6 分，Cronbach α=0.75）"，
不写为什么不二分、为什么不纳入就医方式、合成前做了什么核查。

**统计分析节按发表范例写**：单段或几短段 = 软件版本 + 描述统计指标定义 + 各分析做法（主分析+敏感性分析）+
显著性水平。**禁止** R 包版本号、随机种子、字体嵌入设备、PNG 渲染引擎等工程细节。

自检：
- [ ] 总字数 1200–1800，≥5 个小节
- [ ] 样本量、关键参数与源一致；纳入排除标准清楚；暴露与结局定义可复现
- [ ] 主分析方法与研究目的对应；敏感性分析方法列出
- [ ] **零动机辩解 / 零排除理由 / 零分析前数据核查过程 / 零方法学教科书定义**（grep `因|为了|鉴于|故|是一种.*方法|分析前.*核查|未纳入.*因`，逐条确认不是在论证或科普）
- [ ] 有软件版本、显著性水平、伦理声明
- [ ] 无工程细节、无"先进的/全面系统的"自吹形容词

### Step 5 结果 → `05_results.md`

按研究问题分小节，总字数 1500–2500。**纯客观陈述，不解释原因**（留给讨论）。
- 每段含具体数字；引用表图（"见表1""图1"）；精度与源一致，不私自四舍五入。
- 关键表以 Markdown 表格嵌入正文，关键图以 `![](路径)` 嵌入，不能只写"见表N"。

自检：
- [ ] 总字数 1500–2500，每小节对应一个研究问题，每段 ≥1 个数字
- [ ] 所有数字与源/Table 一致；引用 ≥2 表 ≥1 图，且关键表图已嵌入正文
- [ ] 无解释/推测语（"可能因为""这表明""提示"）；无评价语（"重要意义"）
- [ ] 有序分类变量行序符合 `conventions.R::ORDERED_LEVELS`（低→高）；数字格式统一

### Step 6 讨论 → `06_discussion.md`

5 小节，总字数 1500–2500：
```
## 4 讨论
### 4.1 主要发现（概述，不复制具体数字）
### 4.2 与既往研究比较（具体引用 + 解释异同）
### 4.3 机制/原因分析（谨慎，非确定事实）
### 4.4 优势与局限（每条局限=局限点+影响+应对）
### 4.5 结论与建议（明确结论 + ≥3 条建议）
```

自检：
- [ ] 总字数 1500–2500，5 小节完整
- [ ] 4.1 ≤300 字、不复制结果具体数字；4.2 ≥2 项具体比较；4.2/4.3 `[待补充引用]` ≥4 处
- [ ] 4.4 局限 ≥3 条，每条三要素齐；4.5 有明确结论（非全条件句）+ ≥3 条建议
- [ ] 段首不重复（"本研究"≤2 次）；"综上所述"≤1 次；结论与摘要【结论】一致

#### 结论作独立章节时（学位论文/部分期刊设"5 结论"）

结论是全文收口，火候在"既不单薄、也不臃肿"之间——**一段为宜（期刊论著），用概括性语言**：重申研究做了
什么 → 核心发现（2–4 点，**文字概括**）→ 意义 → 一条审慎的总体建议 → 一句局限与展望。

两端都不要：
- **太少/太空**（反例）："本研究发现照护负担受多因素影响，建议加强支持，未来需进一步研究。"——无信息量。
- **太长/重复数据**（反例）：结论里罗列 β、95%CI、P 值，或把结果与摘要【结论】整段照抄。**结论不重复统计
  量、不复制结果细节**；β/P 留在结果与讨论，结论只给方向性结论。
- **据探索性结果提具体建议**（反例）：未做多重比较校正/未做跨组检验的亚组、交互、分位数结果，**不得**在
  结论升格为确定结论、更不得据此提"对 X 人群侧重 Y"这类高度具体的分层建议（见 review-killers.md）。

自检：结论 1 段、约 150–300 字；核心发现文字概括 2–4 点；**零 β/P/CI 统计量**；与摘要【结论】各有侧重不
大面积重复；建议与证据强度匹配（探索性结果只给"提示性"措辞）；不引入结果里没有的新数据。

### Step 7 参考文献 → `07_references.md`

GB/T 7714-2015 顺序编码制。正文 `[待补充引用]` 处替换为 `[N]` 并在此列出。
- 作者 ≤3 人全列，>3 人前 3 位加"等"（中）/"et al"（英）；中英文文献均列；优先近 5 年。

自检：
- [ ] ≥15 条，中英文均有，格式统一，每条含作者+题名+期刊/出版社+年份+起止页
- [ ] 若用户未供文献，用 `[待补充引用]` 占位并注明"共 N 处需补充"

## 2 国家标准速查

### 2.1 标题编号
阿拉伯数字连续编号，`.` 分隔，最多 4 级，编号后空 1 字写标题：
```
1 引言 / 2 材料与方法 / 2.1 研究设计 / 2.2.1 纳入标准 / 3 结果 / 4 讨论 / 5 结论
```

### 2.2 结构式摘要
`【目的】1–2句 【方法】2–4句 【结果】3–5句含关键数据 【结论】1–2句`；300–500 字；第三人称过去时；独立成文。

### 2.3 参考文献格式（GB/T 7714-2015）
```
期刊: [N] 作者. 题名[J]. 刊名, 年, 卷(期): 起止页码.
书籍: [N] 作者. 书名[M]. 版次. 出版地: 出版社, 年.
报告: [N] 机构. 报告名[R]. 出版地: 出版社, 年.
标准: [N] 标准代号 标准名称[S]. 出版地: 出版社, 年.
网络: [N] 作者. 题名[EB/OL]. (发布日期)[引用日期]. URL.
```

## 3 Word 排版规范

### 3.1 字体字号

| 元素 | 中文字体 | 西文/数字 | 字号 |
|------|----------|-----------|------|
| 论文题目 | 黑体 | Times New Roman | 小二(18pt) |
| 作者姓名 | 宋体 | Times New Roman | 五号(10.5pt) |
| 摘要/关键词标题 | 黑体 | – | 小四(12pt) |
| 摘要/关键词正文 | 楷体_GB2312 | Times New Roman | 五号(10.5pt) |
| 一级标题 | 黑体 | Times New Roman Bold | 四号(14pt) |
| 二级标题 | 黑体 | Times New Roman Bold | 小四(12pt) |
| 三级标题 | 黑体 | Times New Roman | 五号(10.5pt) |
| 正文 | 宋体 | Times New Roman | 五号(10.5pt) |
| 表格内容/参考文献 | 宋体 | Times New Roman | 小五(9pt) |

### 3.2 段落
首行缩进 2 字符；1.5 倍行距；页边距上下 2.54cm、左右 3.17cm；两端对齐。

### 3.3 三线表
表题在表上方；顶线 1.5pt、栏目线 0.5pt、底线 1.5pt；表注在表下方用"注："引导。

## 4 中文期刊投稿适配

> 投稿前以目标期刊**当年官网 Guide for Authors / 投稿须知**为准；下表为常见基线，可能更新。

### 4.1 中国食品卫生杂志

| 项目 | 要求 |
|------|------|
| 中文题名 | ≤20字，无非公认缩略词、无标点、无副标题 |
| 摘要 | 300–400字，结构式四要素 |
| 关键词 | 3–8个，分号分隔 |
| 正文字数 | 一般 ≤5000字（超额加倍收版面费） |
| 图表合计 | ≤6个 |
| 作者信息 | 第一作者：姓名、性别、民族、出生年、职称、学位、电话、E-mail、ORCID |
| 参考文献 | GB/T 7714-2015 顺序编码制，优先近5年 |
| 摘要人称 | 第三人称，不用"本文""作者" |

### 4.2 中华预防医学杂志

| 项目 | 要求 |
|------|------|
| 论著字数 | ≥5000汉字（含图表、摘要、参考文献） |
| 摘要 | ≤500字，结构式 |
| 关键词 | 3–5个，按 CY/T 173-2019 规范 |
| 作者规则 | 1–3名全列，>3名列前3名加"等" |
| 参考文献 | GB/T 7714-2015，每条必著起止页 |
| 计量单位 | 法定计量单位（GB 3100~3102-1993） |

### 4.3 字数交叉校验速查

| 部分 | 目标字数 | 下限告警 | 上限告警 |
|------|----------|----------|----------|
| 摘要 | 300–500 | <280 | >520 |
| 引言 | 800–1200 | <750 | >1300 |
| 方法 | 1200–1800 | <1100 | >1900 |
| 结果 | 1500–2500 | <1400 | >2600 |
| 讨论 | 1500–2500 | <1400 | >2600 |
| 全文 | 5500–7500 | <5000 | >8000 |

写完做交叉一致性校验：同一数据点（样本量、主要估计值、CI）在**摘要=结果=讨论=表图=源**五处必须一致。
